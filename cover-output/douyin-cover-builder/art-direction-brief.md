# Art Direction Brief — AI 的设计灵感库

## 输入（已适配用户约束）
- 主题：介绍给 AI 调用的设计灵感库 Design Resource
- 关键点：灵感站 / 动效 / 组件 / Skills / Web Prompt
- 人物：无（用户明确要求封面不出现人；右下角改用资源目录悬浮卡替代真人位）
- 画幅：竖版（输出 1080×1440，适配抖音信息流封面）

更多示例与对比图：https://github.com/0x00000003/douyin-cover-builder

## 封面类型
概念总览（Overview）——一眼讲清「这是什么、给谁用」。

## HERO 场景隐喻
左下–中下：多块悬浮浏览器窗口叠放（Awwwards / React Bits / Shader Gradient），像被 AI「抽进」一份发光的资源目录；目录本封面写着 `INS · MOT · CMP · SKL`，有细微粒子与网格牵引线连向窗口。

## 配色方案（创意/设计主题）
- 主色：深紫黑 `#1a0b2e` → `#2d1b4e`（背景）
- 辅色：品红紫 `#c084fc`（光晕/边线）
- 高亮色：电光橙 `#ff6b35`（关键词「设计灵感库」）
- 主标题非关键词：明黄 `#ffe566`（系列金黄体系）

## 关键词高亮策略
- 标题拆行：`给 AI 用的` / `设计灵感库`
- 关键词「设计灵感库」用高亮橙 + 更粗描边 + 轻外发光
- 非关键词「给 AI 用的」保持明黄粗描边

## 轻弧排版策略
- 「设计灵感库」一行沿左下→右上极轻弧上扬（不可夸张）
- 底部标签 `灵感站 · 动效 · 组件 · Skills` 沿同向微弧环绕 HERO，不水平死排

## 版式
- 顶部：大标题
- 左下/中下：网站截图 HERO
- 右下：资源目录悬浮卡（替代真人位）
- 安全边距 6–8%
